# **The Laws Every Designer Should Know**

These aren\'t trends. They\'re timeless principles.

## **1. Hick\'s Law**

More choices

↓

Slower decisions

Remove options.

## **2. Fitts\'s Law**

Large targets

Close distance

↓

Faster clicking

CTA buttons should be large.

## **3. Jakob\'s Law**

Users expect your website

to work like every other good website.

Innovation ≠ confusion.

## **4. Miller\'s Law**

People remember limited chunks of information.

Group information.

## **5. Gestalt Principles**

Objects close together

feel related.

Spacing creates meaning.

## **6. Pareto Principle**

20%

of your interface

creates 80%

of the value.

Focus there.

## **7. Serial Position Effect**

People remember

the first

and last

thing most.

Make hero and footer memorable.

## **8. Peak-End Rule**

People judge an experience

by

its peak

and ending.

## **9. Doherty Threshold**

Fast interfaces

feel smarter.

## **10. Von Restorff Effect**

One unique object

gets attention.

Only ONE primary CTA.

# **The Golden Formula**

Before adding **any** element to a screen, ask yourself these five
questions:

1.  **Purpose**: Why does this exist?

2.  **Priority**: Is it the most important thing on the screen?

3.  **Placement**: Is it where users naturally expect it?

4.  **Clarity**: Can someone understand it in under 3 seconds?

5.  **Proof**: Does it help build trust or move users toward the goal?

If an element can\'t justify itself with those questions, it probably
doesn\'t belong.

of professional UX.

# **Portfolio Design Constitution**

### **Version 1.0**

# **Purpose**

This document serves as the design constitution for my personal
portfolio.

It defines the principles, constraints, storytelling structure, writing
style, UX philosophy, and decision-making framework that every design
and development decision must follow.

This portfolio should never resemble a traditional résumé or project
gallery.

Instead, it should function as a product experience that gradually earns
trust and guides visitors toward contacting me.

When conflicts arise between aesthetics and usability, prioritize
usability.

When conflicts arise between creativity and clarity, prioritize clarity.

Every design decision must support the business objective.

# **Primary Goal**

Transform visitors into qualified clients by demonstrating not only my
work but also my thinking, process, professionalism, and ability to
solve business problems through design.

The portfolio should answer these questions before the visitor asks
them:

• Why should I trust you?\
• Why should I hire you?\
• How do you solve problems?\
• Can you communicate clearly?\
• Can you collaborate effectively?\
• What makes you different?\
• What should I do next?

# **Core Philosophy**

This portfolio is not about me.

It is about the client\'s confidence.

The user should never feel like they are reading a résumé.

They should feel like they are experiencing how I think.

Every section must reduce uncertainty.

Every interaction must build trust.

Every page must move users toward one action.

# **Content Rules**

Every section must include:

• Label\
• Narrative Hook\
• Supporting Idea\
• Primary CTA (if applicable)

Never introduce unnecessary content.

If a section does not support the user journey, remove it.

# **Narrative Rules**

The portfolio tells one continuous story.

Every section should naturally lead to the next.

Avoid abrupt topic changes.

Visitors should never wonder why a section exists.

# **Writing Guidelines**

## **Labels**

Purpose

Navigation only.

Rules

• Familiar wording\
• Business terminology\
• Easy to scan\
• Prefer action-oriented language

Examples

Discover

Explore

Build

Create

Improve

Deliver

Collaborate

Launch

## **Narrative Hook**

Purpose

Capture curiosity.

Rules

• One strong statement\
• Maximum one sentence\
• Memorable\
• Outcome-focused\
• Never overloaded with buzzwords

## **Supporting Idea**

Purpose

Support the narrative hook without repeating it.

Rules

• Two to five short phrases\
• Scan-friendly\
• Outcome-driven\
• Avoid long paragraphs

## **Buttons**

Rules

Always begin with an action verb.

Examples

View Work

Explore Projects

See Process

Book Call

Start Project

Contact Me

Download Resume

# **Visual Communication Rules**

Design should communicate before users begin reading.

Hierarchy must always be obvious.

One dominant focal point per section.

Use whitespace intentionally.

Use repetition to establish consistency.

Every component must have a purpose.

Remove decorative elements that do not support comprehension.

# **UX Principles**

Always optimize for:

Recognition over recall

Scanning over reading

Progressive disclosure

Clear visual hierarchy

Minimal cognitive load

Accessibility

Fast decision-making

# **FAQ Philosophy**

The FAQ is not customer support.

It is a trust-building strategy.

Its purpose is to answer interview questions before interviews happen.

Every answer should reveal:

• Process\
• Professionalism\
• Decision-making\
• Collaboration style\
• Business thinking

The FAQ should reduce objections while increasing confidence.

# **AI Instructions**

When generating designs or content:

Never generate generic portfolio sections.

Never use résumé language.

Never use filler text.

Never prioritize aesthetics over usability.

Every suggestion must support the user journey.

Every headline should communicate value.

Every supporting element should reinforce the headline.

Avoid unnecessary explanations.

Favor concise, business-oriented communication.

When unsure, simplify.

# **Decision Checklist**

Before adding any section, ask:

Does it build trust?

Does it answer a client question?

Does it support the narrative?

Does it move the user toward the next stage?

Can users understand it within five seconds?

If the answer is \"No\" to any of these questions, redesign or remove
it.

# **Notes for Future Improvements**

This document is intended to evolve.

As the portfolio grows through user testing and client feedback, refine
this constitution without changing its core philosophy.

Innovation is encouraged.

Confusion is not.

Creativity is encouraged.

Clarity is mandatory.
